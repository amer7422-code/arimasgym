# Gym Engine Backend — نظام مصادقة حقيقي

باك اند حقيقي (وليس محاكاة) لتسجيل الدخول وإنشاء الحسابات، مبني بـ Node.js + Express،
مع كلمات مرور مُشفّرة (bcrypt) وجلسات موقّعة بـ JWT، وقاعدة بيانات SQLite حقيقية (ملف واحد).

## 1) التشغيل على جهازك

```bash
cd gym-backend
npm install
cp .env.example .env
# افتح .env وغيّر JWT_SECRET لقيمة عشوائية طويلة
npm start
```

السيرفر سيعمل على: `http://localhost:3000`

## 2) نقاط الاتصال (API Endpoints)

|Method|Endpoint          |الوصف                                                               |
|------|------------------|--------------------------------------------------------------------|
|POST  |/api/auth/register|إنشاء حساب: `{ name, phone, email?, password }`                     |
|POST  |/api/auth/login   |تسجيل دخول: `{ phone, password }`                                   |
|GET   |/api/auth/me      |بيانات المستخدم الحالي (يحتاج Header: `Authorization: Bearer TOKEN`)|
|GET   |/api/health       |فحص أن السيرفر شغال                                                 |

رقم الهاتف لازم يكون مصري صحيح (01 + 9 أرقام)، وكلمة المرور 6 أحرف على الأقل.

## 3) ربطه بملف الموقع

في ملف `gym-engine-app.html` تم إضافة متغير في أول السكربت:

```js
const API_BASE_URL = 'http://localhost:3000';
```

- أثناء التطوير على جهازك: اتركه كما هو وشغّل الباك اند بـ `npm start`.
- بعد النشر (الخطوة التالية): غيّر القيمة لرابط السيرفر الحقيقي بعد رفعه.

## 4) نشر الباك اند على الإنترنت (مهم — لازم يتم من جهازك)

⚠️ ملاحظة صريحة: أنا لا أملك القدرة على رفع سيرفر فعلي على الإنترنت من هنا (لا يوجد اتصال إنترنت
في بيئة التنفيذ الخاصة بي). الكود المرفق حقيقي 100% ويعمل فعلياً، لكن تشغيله بشكل دائم على رابط
عام يحتاج استضافة. أسهل خيارات مجانية/رخيصة:

1. **Render.com** — ارفع مجلد `gym-backend` على GitHub ثم اربطه بـ Render كـ “Web Service”
   (Build: `npm install`, Start: `npm start`)، وضِف متغيرات البيئة من `.env.example`.
1. **Railway.app** — نفس الفكرة، أسهل في الإعداد.
1. **Fly.io / VPS خاص** — لو محتاج تحكم كامل.

بعد النشر هتاخد رابط زي `https://your-app.onrender.com`، وتحطه في `API_BASE_URL` داخل ملف الـ HTML.

## 5) ملاحظات أمان قبل الاستخدام الفعلي مع عملاء حقيقيين

- غيّر `JWT_SECRET` لقيمة عشوائية طويلة (مش القيمة الافتراضية).
- حدد `CORS_ORIGIN` برابط موقعك بدل `*`.
- فعّل HTTPS (تلقائي على Render/Railway).
- SQLite ممتاز للبداية والتجربة؛ لو حجم البيانات كبر واحتجت أداء أعلى وتزامن كتابة كثيف،
  انقل لـ PostgreSQL لاحقاً — بنية الكود بسيطة وتسمح بذلك بسهولة.
